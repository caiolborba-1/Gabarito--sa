        O QUE É UM GRUD?
        
         O CRUD É UM ACRÔNIMO PARA AS OPERAÇÕES FUNDAMENTAIS EM QUALQUER SISTEMA, QUE INTERAGE COM O BANCO DE DADOS: CREATE, READ, UPDATE, DELETE. <br>
        ESSAS 4 OPERAÇÕES PERMITEM QUE OS USUÁRIOS POSSAM CRIAR REGISTROS NOVOS, <br> LER OU RECUPERAR DADOS EXISTENTES, ATUALIZAR INFORMAÇÕES PREVIAMENtE ARMAZENADAS, <br> E FINALMENTE EXCLUIR REGISTROS QUANDO NECESSÁRIOS.
   
  

        CREATE

        ESTA ETAPA ENVOLVE A CRIAÇÃO DE NOVOS DADOS EM UM SISTEMA. POR EXEMPLO, QUANDO UM USUÁRIO SE REGISTRA EM UM SITE, UM NOVO REGISTRO É CRIADO EM UM BANCO DE DADOS. <br>
        ESSA FUNÇÃO É FUNDAMENTAL PARA INICIAR QUALQUER INTERAÇÃO EM SISTEMAS QUE DEPENDEM DE DADOS, <br>
        SEJA EM UM SITE DE E-COMMERCE AO ADICIONAR UM NOVO PRODUTO OU EM UM SISTEMA DE GERENCIAMENTO DE USUÁRIOS AO CRIAR UM NOVO PERFIL.
   


        Read

        A FUNÇÃO “READ” É O QUE PERMITE AOS SISTEMAS ACESSAR E EXIBIR DADOS. <br>
        ELA É USADA CADA VEZ QUE SOLICITAMOS INFORMAÇÕES, COMO AO VERIFICAR O SALDO EM UMA CONTA BANCÁRIA ONLINE. 



        Update
        
        “UPDATE” REFERE-SE À ATUALIZAÇÃO DE DADOS EXISTENTES. <br>
        ESTA FUNÇÃO É CRUCIAL PARA MANTER A RELEVÂNCIA E A PRECISÃO DAS INFORMAÇÕES EM UM BANCO DE DADOS.



        Delete

        ESTA FUNÇÃO É USADA PARA REMOVER DADOS DE UM SISTEMA. <br>
        ELA DEVE SER MANUSEADA COM CUIDADO, POIS UMA VEZ QUE OS DADOS SÃO APAGADOS, PODEM NÃO SER RECUPERÁVEIS. 



        Crud em PHP

        (CREATE, READ, UPDATE, DELETE) PERMITE CRIAR, LER, ATUALIZAR E EXCLUIR DADOS EM UM BANCO MYSQL, USANDO COMANDOS SQL COMO INSERT, SELECT, UPDATE, E DELETE. <br>
        A ESTRUTURA BÁSICA ENVOLVE ARQUIVOS PARA CONEXÃO (CONFIG.PHP) LISTAGEM (INDEX.PHP) , FORMULÁRIOS DE CADASTRO/EDIÇÃO E SCRIPTS DE PROCESSAMENTO DE AÇÕES. <br>
        O USO DE PDO OU MYSQL GARANTE A INTERAÇÃO SEGURA COM O BANCO DE DADOS.


        Aplicação do CRUD no Projeto

        No projeto desenvolvido, o CRUD será utilizado para gerenciar os dados do sistema ferroviário com sensores IoT.

        As operações vão funcionar da seguinte forma:
        Create: cadastrar sensores, locomotivas e novos registros de monitoramento.
        Read: visualizar dados como temperatura, velocidade, pressão e localização dos trens.
        Update: atualizar informações de sensores ou dados cadastrados.
        Delete: remover registros antigos ou sensores desativados.

        Dessa forma o crud permite o controle das informações do sistema, sempre garantindo organização, monitoramento e gerenciamento dos dados.
